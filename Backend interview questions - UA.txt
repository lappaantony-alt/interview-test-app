Git
1. Що таке Git?
 Git – це розподілена система керування версіями, яка дозволяє відслідковувати зміни в коді та координувати роботу між розробниками.

2. Які основні команди Git ви знаєте?
 Найпоширеніші команди – git init, git clone, git add, git commit, git push, git pull, git branch, git merge, git status та git log.

3. Що таке commit?
 Commit – це збереження поточного стану змін у локальному репозиторії з відповідним повідомленням, що описує зроблені зміни.

4. Як повернутися до попереднього commit?
 Можна використати команди git revert (створює новий commit, що скасовує зміни) або git reset (змінює історію локально, що може бути ризиковано).

5. Що таке branch (гілка) в Git?
 Гілка – це окрема лінія розробки, що дозволяє ізолювати нові функції або виправлення від основної гілки.

6. Як об’єднати гілки?
 Об’єднання гілок здійснюється за допомогою команди git merge, яка інтегрує зміни однієї гілки в іншу.

7. Що таке merge conflict?
 Merge conflict виникає, коли дві гілки містять суперечливі зміни в одному й тому ж рядку файлу. Рішення конфлікту вимагає ручного втручання.

8. У чому різниця між Git fetch і Git pull?
 git fetch завантажує останні зміни з віддаленого репозиторію, але не об’єднує їх з локальною гілкою, тоді як git pull одночасно завантажує і виконує merge.

9. Що таке rebase у Git?
 Rebase – це операція, що дозволяє «переписати» історію змін, базуючи поточну гілку на іншій, забезпечуючи більш лінійну історію комітів.

10. Які переваги та недоліки використання rebase?
 Переваги – чиста історія комітів, легше відслідковувати зміни;
Недоліки – зміна історії, що може ускладнити співпрацю, якщо коміти вже опубліковані.

Maven
   11. Що таке Maven?
 Maven – це інструмент для управління проєктами, який автоматизує процес побудови, управління залежностями та розгортання Java-додатків.

   12. Які основні компоненти Maven?
 Основні компоненти – POM (Project Object Model), репозиторії, цикли побудови (lifecycle) та плагіни.

   13. Що таке POM?
 POM – це XML-файл (pom.xml), який описує конфігурацію проєкту, залежності, плагіни та інші налаштування.

   14. Як працює система залежностей у Maven?
 Maven автоматично завантажує потрібні бібліотеки з віддалених репозиторіїв згідно з описом у POM, а також вирішує їх транзитивні залежності.

   15. Що таке lifecycle у Maven?
 Lifecycle – це набір фаз (наприклад, validate, compile, test, package, install, deploy), через які проходить проєкт під час побудови.

   16. Як додати залежність у Maven?
 Залежність додається у секцію <dependencies> файлу pom.xml із зазначенням groupId, artifactId та версії.

   17. Що таке plugin у Maven?
 Plugin – це набір інструментів, який розширює функціональність Maven (наприклад, для компіляції, тестування чи створення архівів).

   18. Як запустити Maven build?
 Maven build запускається через командний рядок за допомогою команди mvn clean install або іншої відповідної команди залежно від циклу побудови.

   19. Що таке dependency management у Maven?
 Dependency management дозволяє централізовано керувати версіями залежностей, визначаючи їх у батьківському POM для використання в підпроєктах.

   20. Що таке Maven Repository і які існують типи репозиторіїв?
 Репозиторій – це сховище артефактів. Основні типи – локальний (на машині розробника), віддалений (наприклад, Maven Central) і корпоративний.

   21. Які команди Maven використовуються для тестування проєкту?
 Основна команда для запуску тестів – mvn test. Також існує команда mvn verify, що включає додаткові перевірки.

   22. Які аналоги Maven вам відомі?
 Gradle, Ant, SBT.

Java Basics
      23. Що таке JVM?
 JVM (Java Virtual Machine) – це віртуальна машина, яка виконує байт-код Java та забезпечує кросплатформеність.

      24. Що таке JRE та JDK?
 JRE (Java Runtime Environment) містить JVM та бібліотеки для виконання програм, а JDK (Java Development Kit) включає JRE та інструменти для розробки (компілятор, дебаггер тощо).

      25. Що таке bytecode?
 Bytecode – це проміжний код, який генерується компілятором Java і виконується JVM.

      26. Що таке клас?
 Клас – це шаблон або блакитний принт, що описує властивості (поля) та поведінку (методи) об’єктів.

      27. Що таке об’єкт?
 Об’єкт – це екземпляр класу, який має конкретний стан (значення полів) та може виконувати методи класу.

      28. Що таке змінна?
 Змінна – це іменована область пам’яті, що зберігає значення певного типу даних.

      29. Які основні типи даних у Java?
 Примітивні типи – byte, short, int, long, float, double, char, boolean; а також об’єктні типи (наприклад, String, Integer).

      30. Що таке оператори в Java?
 Оператори – це символи або комбінації символів, що виконують операції над даними (арифметичні, логічні, порівняння тощо).

      31. Що таке метод?
 Метод – це блок коду, що виконує певну функцію, може приймати параметри і повертати значення.

      32. Як створити метод у Java?
 Метод створюється з оголошенням модифікатора доступу, типу повернення, імені методу та списку параметрів, наприклад:

public int sum(int a, int b) {
                    return a + b;
}
         33. Що таке інкапсуляція?
 Інкапсуляція – це принцип ООП, який приховує внутрішню реалізацію класу та забезпечує доступ до даних через методи.

         34. Що таке пакет (package) у Java?
 Пакет – це група пов’язаних класів, що організовує структуру проєкту і допомагає уникнути конфліктів імен.

         35. Як працюють модифікатори доступу?
 Модифікатори доступу (public, protected, private, package-private) визначають рівень доступу до класів, полів та методів.

         36. Що таке статичний метод та поле?
 Статичний метод або поле належать класу, а не конкретному об’єкту, і доступні без створення екземпляру класу.

         37. Що таке конструктор?
 Конструктор – це спеціальний метод, який викликається при створенні нового об’єкту і ініціалізує його поля.

         38. Що таке перевантаження методів?
 Перевантаження методів – це визначення декількох методів з однаковим ім’ям, але різними параметрами.

         39. Що таке рекурсія?
 Рекурсія – це процес, коли метод викликає сам себе для вирішення підзадачі, поки не буде досягнута умова завершення.

         40. Що таке винятки (exceptions) в Java?
 Винятки – це механізм обробки помилок, що дозволяє передавати інформацію про помилки у виконанні програми та коректно їх обробляти за допомогою try-catch блоків.

         41. Яка різниця між checked та unchecked винятками?
 Checked винятки перевіряються компілятором і мають бути оброблені або оголошені у сигнатурі методу, тоді як unchecked винятки (RuntimeException) – ні.

         42. Що таке generics у Java?
 Generics дозволяють параметризувати класи та методи, забезпечуючи типобезпеку під час компіляції та зменшуючи потребу в приведенні типів.

         43. Як працює механізм автопакування (autoboxing)?
 Автопакування автоматично перетворює примітивні типи у відповідні об’єктні (наприклад, int в Integer) та навпаки.

         44. Що таке enum в Java?
 Enum – це спеціальний тип класу, що визначає набір констант, часто використовується для представлення обмеженого набору значень.

         45. Як працює система класових завантажувачів (class loaders)?
 Class loaders відповідають за динамічне завантаження класів під час виконання програми, розділяючи простір імен для уникнення конфліктів.

         46. Що таке пакетна декларація (package declaration) та чому вона важлива?
 Декларація пакету вказує, до якого простору імен належить клас, що допомагає організувати код і уникнути конфліктів імен.

         47. Що таке анотації (annotations) в Java?
 Анотації – це спеціальні метадані, які можна додавати до класів, методів або полів для надання додаткової інформації, що використовується компілятором або фреймворками.

         48. Як оголосити і використовувати власну анотацію?
 Власну анотацію оголошують за допомогою @interface, а потім її можна застосовувати над класами, методами чи полями.

Java OOP
            49. Що таке об’єктноорієнтоване програмування (ООП)?
 ООП – це парадигма програмування, заснована на поняттях класів і об’єктів, що дозволяє моделювати реальний світ.

            50. Які основні принципи ООП?
 Основні принципи – інкапсуляція, наслідування, поліморфізм і абстракція.

            51. Що таке наслідування?
 Наслідування дозволяє одному класу (підклас) отримувати властивості та методи іншого класу (суперклас), сприяючи повторному використанню коду.

            52. Що таке поліморфізм?
 Поліморфізм дозволяє об’єктам різних класів обробляти запити через один інтерфейс, що дозволяє використовувати один і той же код для різних типів.

            53. Що таке абстракція?
 Абстракція – це принцип виділення суттєвих характеристик об’єкта, приховуючи деталі реалізації.

            54. Що таке інтерфейс?
 Інтерфейс – це контракт, що оголошує набір методів, які клас повинен реалізувати, без опису їх реалізації.

            55. Як використовувати абстрактні класи?
 Абстрактні класи використовуються для визначення базової логіки, яку наслідують підкласи, та можуть містити як абстрактні, так і реалізовані методи.

            56. Чим інтерфейс від абстрактного класу?
 Інтерфейс не містить реалізації методів (до Java 8), а абстрактний клас може містити як реалізовані, так і абстрактні методи; клас може реалізувати декілька інтерфейсів, але наслідувати лише один абстрактний клас.

            57. Що таке інтерфейс маркер (marker interface) і навіщо він потрібен?
 Маркерний інтерфейс не містить методів, а лише позначає клас як такий, що має певні властивості (наприклад, Serializable).

            58. Що таке default методи в інтерфейсах?
 Default методи – це методи з реалізацією в інтерфейсах, що дозволяють розширювати інтерфейси без руйнування існуючих реалізацій.

            59. Як працюють статичні методи в інтерфейсах?
 Статичні методи в інтерфейсах можуть бути викликані без створення екземпляру інтерфейсу, забезпечуючи допоміжні функції.

            60. Що таке композиція?
 Композиція – це принцип, коли об’єкт містить інші об’єкти як свої поля для досягнення складнішої поведінки.

            61. Чим відрізняється композиція від наслідування?
 Наслідування дозволяє розширити поведінку класу шляхом спадкування, тоді як композиція полягає в побудові нової функціональності через включення інших об’єктів.

            62. Що таке агрегація?
 Агрегація – це тип відносин, коли один об’єкт є частиною іншого, але може існувати незалежно від нього.

            63. Як працює механізм перевантаження методів?
 Перевантаження методів дозволяє визначати декілька методів з однаковим ім’ям, але з різними типами або кількістю параметрів.

            64. Що таке перевизначення методів?
 Перевизначення (override) – це процес, коли підклас реалізує власну версію методу, який уже існує в суперкласі.

            65. Які модифікатори доступу використовуються в ООП?
 Найчастіше використовуються public, protected, private та package-private (без явного модифікатора).

            66. Що таке final клас?
 Final клас не може бути наслідуваним, тобто інші класи не можуть розширити його функціонал.

            67. Що таке static?
 Статичні поля та методи належать класу, а не конкретним об’єктам, що дозволяє отримувати до них доступ без створення екземпляру.

            68. Як викликати конструктор суперкласу?
 Використовують ключове слово super у конструкторі підкласу для виклику конструктора суперкласу.

            69. Що таке encapsulation і чому воно важливе?
 Інкапсуляція дозволяє приховувати внутрішню логіку класу, надаючи доступ лише через публічні методи, що сприяє безпечному та контрольованому доступу до даних.

            70. Що таке SOLID принципи в ООП?
 SOLID – це набір принципів дизайну (Single responsibility, Open/closed, Liskov substitution, Interface segregation, Dependency inversion), що допомагають створювати зрозумілий, підтримуваний та масштабований код.

            71. Як забезпечити іммутабельність (immutable) об’єкта в Java?
 Забезпечують шляхом оголошення класу як final, використанням приватних фінальних полів, а також відсутністю setter-методів.

Java Collections
               72. Що таке Java Collections Framework?
 Це набір інтерфейсів та класів, що забезпечують структури даних (списки, множини, мапи тощо) та алгоритми для роботи з ними.

               73. Які основні інтерфейси входять до Collections Framework?
 Основні інтерфейси – Collection, List, Set, Queue та Map.

               74. Що таке List?
 List – це колекція, що зберігає елементи у визначеному порядку та допускає повторювані значення.

               75. Що таке Set?
 Set – це колекція, яка зберігає лише унікальні елементи без визначеного порядку.

               76. Що таке Map?
 Map – це колекція, що зберігає пари ключ-значення, де кожен ключ є унікальним.

               77. Які особливості має ArrayList?
 ArrayList забезпечує динамічний масив з можливістю швидкого доступу за індексом, але операції вставки/видалення можуть бути повільними.

               78. Чим відрізняється LinkedList від ArrayList?
 LinkedList базується на зв’язному списку, що дозволяє швидко вставляти/видаляти елементи, а ArrayList забезпечує швидкий доступ за індексом завдяки використанню масиву.

               79. Що таке HashSet?
 HashSet – це реалізація інтерфейсу Set, яка використовує хеш-таблицю для забезпечення унікальності елементів та швидкого пошуку.

               80. Як працює TreeSet?
 TreeSet зберігає елементи у відсортованому порядку, використовуючи червоно-чорне дерево, що дозволяє швидкий пошук та навігацію.

               81. Що таке HashMap?
 HashMap – це реалізація інтерфейсу Map, яка зберігає дані у вигляді пар ключ-значення з використанням хешування для швидкого доступу.

               82. Чим TreeMap відрізняється від HashMap?
 TreeMap зберігає записи у відсортованому порядку ключів, а HashMap – у випадковому порядку, забезпечуючи кращу продуктивність при великій кількості елементів.

               83. Що таке Iterator?
 Iterator – це інтерфейс для послідовного обходу елементів колекції без розкриття її внутрішньої структури.

               84. Як використовувати цикл foreach у Java?
 Цикл foreach дозволяє пройтися по кожному елементу колекції, наприклад:
for (String item : list) {
                    System.out.println(item);
}




                  85. Що таке Concurrent Collections?
 Concurrent Collections – це колекції, розроблені для роботи в багатопоточному середовищі, забезпечуючи безпечний одночасний доступ.

                  86. Як працюють методи equals() і hashCode() в колекціях?
 Методи equals() порівнюють об’єкти на рівність, а hashCode() повертає числове значення, що використовується для розподілу об’єктів у хеш-структурах. Їх правильна реалізація є критично важливою для коректної роботи колекцій.

Functional Interfaces
                     87. Що таке функціональний інтерфейс?
 Функціональний інтерфейс – це інтерфейс, який містить лише один абстрактний метод і може використовуватися для лямбда-виразів.

                     88. Які вимоги до функціонального інтерфейсу?
 Він повинен мати лише один абстрактний метод; додатково можна використовувати default та static методи.

                     89. Наведіть приклад функціонального інтерфейсу у Java.
 Приклад – інтерфейс Runnable, який містить єдиний метод run().

                     90. Як використовувати лямбда-вирази з функціональними інтерфейсами?
 Лямбда-вирази дозволяють реалізувати єдиний метод функціонального інтерфейсу коротким синтаксисом, наприклад:

Runnable r = () -> System.out.println("Hello");

                     91. Що таке метод посилання (method reference)?
 Це скорочена форма лямбда-виразу, яка посилається на вже існуючий метод, наприклад: System.out::println.

Streams
                        92. Що таке Streams в Java?
 Streams – це API для роботи з послідовностями даних, що дозволяє обробляти колекції декларативно (через операції map, filter, reduce тощо).

                        93. Як створити потік з колекції?
 За допомогою методу .stream(), наприклад, list.stream().

                        94. Що таке операції intermediate?
 Це проміжні операції (filter, map, sorted), які повертають новий потік та дозволяють створювати ланцюжки обробки.

                        95. Що таке terminal operations?
 Terminal operations (collect, forEach, reduce) завершують обробку потоку і повертають результат.

                        96. Наведіть приклад використання map() операції.

List<Integer> nums = Arrays.asList(1, 2, 3);
List<Integer> squares = nums.stream()
                    .map(n -> n * n)
                    .collect(Collectors.toList());
                           97. Як використовувати filter() в Stream API?
 Метод filter() дозволяє відфільтрувати елементи потоку за певною умовою, наприклад: stream.filter(x -> x > 10).

                           98. Що таке reduce()?
 Reduce – це terminal операція, яка об’єднує елементи потоку в одне значення за допомогою заданої операції.

                           99. Які переваги використання Stream API?
 Stream API спрощує обробку даних, робить код більш декларативним, дозволяє ефективно використовувати паралельну обробку та покращує читабельність.

                           100. Як працює parallelStream()?
 Метод parallelStream() створює паралельний потік, що розподіляє обробку елементів між декількома потоками для підвищення продуктивності.

                           101. Що таке lazy evaluation у Streams?
 Lazy evaluation означає, що операції на потоці не виконуються доти, доки не буде викликана terminal operation.

                           102. Як сортувати дані за допомогою Stream API?
 За допомогою операції sorted(), наприклад: stream.sorted() або з компаратором: stream.sorted(Comparator.naturalOrder()).

                           103. Що таке flatMap() і для чого воно використовується?
 FlatMap() дозволяє об’єднати кілька потоків в один, розгладжуючи вкладені структури даних.

                           104. Як працює collect() у Stream API?
 Метод collect() акумулює результати потоку в колекцію або інший контейнер, наприклад, Collectors.toList().

                           105. Наведіть приклад обробки числових потоків.

IntStream.range(1, 10)
        .filter(n -> n % 2 == 0)
        .forEach(System.out::println);

                           106. Як використовувати метод peek() в Stream API?
 Метод peek() дозволяє виконувати проміжні дії (наприклад, логування) без зміни елементів потоку, наприклад: stream.peek(System.out::println).

Optional
                              107. Що таке Optional в Java?
 Optional – це контейнер, який може містити або не містити значення, запобігаючи NullPointerException.

                              108. Як створити об’єкт Optional?
 За допомогою методів Optional.of(value), Optional.ofNullable(value) або Optional.empty().

                              109. Які методи Optional використовуються для перевірки наявності значення?
 Методи isPresent(), ifPresent() та orElse() дозволяють перевірити та обробити значення.

                              110. Як використовувати Optional.orElseThrow()?
 Метод orElseThrow повертає значення, якщо воно є, або кидає виняток, який задається через функціональний інтерфейс Supplier.

                              111. Що таке Optional.ifPresentOrElse()?
 Цей метод приймає два параметри – споживача для наявного значення та Runnable для дій, якщо значення відсутнє.

Databases
                                 112. Які основні типи баз даних ви знаєте?
 Реляційні, нереляційні (NoSQL), графові, документоорієнтовані, колоночні тощо.

                                 113. Що таке реляційна база даних?
 Це база даних, що організована у вигляді таблиць із зв’язками між ними, де дані зберігаються у вигляді записів і стовпців.

                                 114. Які системи керування базами даних ви знаєте?
 Найбільш відомі – MySQL, PostgreSQL, Oracle, Microsoft SQL Server, MongoDB (як приклад NoSQL).

                                 115. Що таке транзакція в базі даних?
 Транзакція – це послідовність операцій, що виконуються як єдине ціле, з можливістю відкату у разі помилки.

                                 116. Як забезпечується цілісність даних?
 За допомогою ключів, обмежень (constraints), транзакцій та механізмів ACID (атомарність, консистентність, ізольованість, довговічність).

                                 117. Що таке нормалізація баз даних і які рівні нормалізації існують?
 Нормалізація – це процес оптимізації структури бази даних для усунення надлишковості, зазвичай описується нормальними формами від 1NF до 5NF.

                                 118. Що таке денормалізація і коли її використовують?
 Денормалізація – це навмисне відхилення від нормальних форм для підвищення продуктивності читання, хоча може призвести до надлишковості даних.

                                 119. Як працюють індекси в реляційних базах даних?
 Індекси прискорюють пошук даних, зберігаючи посилання на рядки таблиці, але можуть уповільнювати операції запису.

SQL
                                    120. Що таке SQL?
 SQL (Structured Query Language) – це мова запитів для взаємодії з реляційними базами даних.

                                    121. Як написати базовий SELECT запит?
 SELECT * FROM table_name;

                                    122. Що таке JOIN в SQL?
 JOIN – це операція, що дозволяє об’єднати рядки з двох або більше таблиць на основі спільного поля.

                                    123. Які типи JOIN існують?
 Найпоширеніші – INNER JOIN, LEFT JOIN, RIGHT JOIN, FULL OUTER JOIN.

                                    124. Що таке агрегатні функції в SQL?
 Агрегатні функції (SUM, AVG, COUNT, MIN, MAX) виконують обчислення над набором рядків і повертають одне значення.

                                    125. В чому різниця між DELETE, DROP та TRUNCATE?

                                       * DELETE: Видаляє рядки з таблиці на основі умови (може бути відкотлено у межах транзакції у багатьох СКБД).
                                       * DROP: Повністю видаляє об’єкт бази даних, наприклад таблицю чи базу.
                                       * TRUNCATE: Швидко видаляє всі рядки з таблиці та часто не підлягає відкату.

Spring Boot
                                       126. Що таке Spring Boot?
 Spring Boot – це фреймворк для створення Spring-додатків із мінімальними налаштуваннями завдяки автоконфігурації та готовим шаблонам.

                                       127. Які переваги Spring Boot?
 Серед переваг – швидкий старт, мінімум налаштувань, автоконфігурація, вбудований сервер, готовність до розгортання та розширюваність.

                                       128. Як створити Spring Boot додаток?
 Можна скористатися Spring Initializr (start.spring.io) або створити проєкт вручну, налаштувавши POM та класи конфігурації.

                                       129. Що таке автоконфігурація?
 Автоконфігурація автоматично налаштовує компоненти Spring залежно від наявних залежностей і конфігурацій у проєкті.

                                       130. Як працює Spring Initializr?
 Spring Initializr – це веб-сервіс, який генерує шаблон Spring Boot проєкту з вибраними залежностями та налаштуваннями.

                                       131. Що таке starter dependency?
 Стартер залежності – це набір зручних залежностей, що спрощують підключення певних технологій у Spring Boot проєкті.

                                       132. Як налаштовуються властивості додатку в Spring Boot?
 Властивості задаються у файлах application.properties або application.yml, де можна встановлювати налаштування додатку.

                                       133. Що таке Spring Boot Actuator?
 Actuator надає готові кінцеві точки для моніторингу, управління і аналізу роботи Spring Boot додатків.

                                       134. Як використовувати профілі у Spring Boot?
 Профілі дозволяють створювати окремі конфігурації для різних середовищ (dev, prod) через файл налаштувань або анотацію @Profile.

HTTP
                                          135. Що таке HTTP?
 HTTP (Hypertext Transfer Protocol) – це протокол для передачі даних між клієнтом і сервером у вебі.

                                          136. Які основні методи HTTP ви знаєте?
 Основні методи – GET, POST, PUT, DELETE, PATCH, OPTIONS та HEAD.

                                          137. Що таке RESTful API?
 RESTful API – це архітектурний стиль веб-сервісів, який використовує HTTP методи для роботи з ресурсами, ідентифікованими URL.

                                          138. Що таке статус-коди HTTP?
 Статус-коди HTTP – це числові коди, що повертаються сервером для позначення результату запиту (наприклад, 200 OK, 404 Not Found, 500 Internal Server Error).

                                          139. Як працює протокол HTTPS?
 HTTPS – це безпечна версія HTTP, що використовує SSL/TLS для шифрування даних, що передаються між клієнтом і сервером.

                                          140. Що таке cookies і як вони використовуються у вебі?
 Cookies – це невеликі дані, які веб-сервер зберігає на клієнті для ідентифікації користувача, збереження стану сесії та інших параметрів.

Spring Data
                                             141. Що таке Spring Data?
 Spring Data – це частина Spring фреймворку, що спрощує роботу з базами даних шляхом абстрагування рутинних операцій CRUD.

                                             142. Які основні компоненти Spring Data?
 Основні компоненти – репозиторії, шаблони доступу до даних (JdbcTemplate, MongoTemplate) та механізми автоматичного створення реалізацій CRUD.

                                             143. Як працює репозиторій у Spring Data?
 Репозиторій – це інтерфейс, що розширюється від Spring Data, який автоматично генерує реалізацію для стандартних операцій над сутністю (збереження, пошук, видалення).

                                             144. Що таке CRUD операції у Spring Data?
 CRUD – це абревіатура для операцій Create, Read, Update та Delete, які забезпечують базову роботу з даними.

                                             145. Що таке Inversion of Control (IoC) і як вона реалізована у Spring?
 IoC – це принцип, коли створення та управління об’єктами делегуються контейнеру. У Spring це реалізовано через Dependency Injection, що дозволяє автоматично підставляти залежності у біні.

                                             146. Що таке Dependency Injection (DI) і які його типи існують у Spring?
 DI – це механізм, що забезпечує автоматичне передання залежностей (об’єктів) до бінів. Основні типи – конструкторна ін’єкція, сеттерна ін’єкція та ін’єкція через поля.

                                             147. Які існують області (scopes) бінів у Spring і як вони впливають на життєвий цикл об’єктів?
 Основні області – singleton (один екземпляр на контейнер), prototype (новий екземпляр щоразу), request, session та application. Вони визначають, скільки екземплярів створюється та як довго вони існують.

                                             148. Як інтегрується Spring Security для забезпечення автентифікації та авторизації?
 Spring Security надає комплексний механізм захисту, який включає фільтри для обробки аутентифікації, конфігурацію доступу до ресурсів за допомогою URL-патернів, а також підтримку різних методів автентифікації (Basic, OAuth2, JWT тощо).
